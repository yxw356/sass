> 适合第一次接触本项目的同学，按顺序做即可。

## 一、这个项目是干什么的？

你可以把它理解成：**先往系统里塞文档，再向它提问，它会根据文档内容回答，并告诉你引用了哪一段。**

例如：

1. 上传公司制度 PDF、产品说明 Markdown、会议记录 TXT  
2. 问：「试用期有多久？」「产品支持哪些格式？」  
3. 系统从已上传文档里检索相关内容，用大模型组织成答案，并附上引用片段  

技术名词（了解即可）：

| 名词 | 通俗解释 |
|------|----------|
| **RAG** | 检索 + 生成：先找文档片段，再让 AI 回答 |
| **Embedding** | 把文字变成「向量」，方便相似度搜索 |
| **vLLM** | 本地跑大模型的服务，本项目的「大脑」 |
| **Chroma** | 存在你电脑上的向量数据库 |

## 二、整体架构（一张图）

```
┌─────────────┐     ┌──────────────────┐     ┌─────────────┐
│  浏览器      │────▶│  KB Agent :8080  │────▶│  vLLM :8000 │
│  上传/提问   │     │  FastAPI         │     │  大模型推理  │
└─────────────┘     └────────┬─────────┘     └─────────────┘
                             │
                    ┌────────┴────────┐
                    ▼                 ▼
            data/uploads/      chroma_db/
            （原始文件）         （向量索引）
                    │
                    ▼
            models/bge-small-zh-v1.5/
            （Embedding 模型，本地）
```

**两个端口别搞混：**

| 端口 | 服务 | 作用 |
|------|------|------|
| **8000** | vLLM | 生成回答 |
| **8080** | KB Agent | 上传文档、检索、网页界面 |

## 三、部署前准备

### 3.1 硬件与系统

- **系统**：Linux（推荐）或 macOS；Windows 建议用 WSL2  
- **Python**：3.10 或以上（执行 `python3 --version` 检查）  
- **磁盘**：至少约 **5GB** 空闲（依赖 + Embedding 模型 + 向量库）  
- **内存**：视 vLLM 模型而定；35B 级模型通常需要较大显存/内存  

### 3.2 需要提前有的东西

1. **vLLM 已能正常启动**（见下文第四节）  
2. **能访问终端**（SSH 或本机 Terminal）  
3. **（可选）** 若访问不了 huggingface.co，需用镜像预下载 Embedding 模型

## 四、从零部署（逐步操作）

### 第 1 步：获取代码

```bash
git clone https://github.com/yxw356/sass.git
cd sass
```

若你本地已有项目目录，进入该目录即可，例如：

```bash
cd ~/kb-agent   # 或你的实际路径
```

### 第 2 步：创建 Python 虚拟环境并安装依赖

```bash
python3 -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

看到 `Successfully installed` 即表示依赖安装完成。

### 第 3 步：下载 Embedding 模型（重要）

首次上传文档需要 **Embedding 模型**。国内网络建议用镜像：

```bash
source .venv/bin/activate
HF_ENDPOINT=https://hf-mirror.com python scripts/download_embedding_model.py
```

完成后应存在目录：`models/bge-small-zh-v1.5/`。

> **跳过此步会怎样？** 上传时可能报错 `Network is unreachable` 或 `文档索引失败`。

### 第 4 步：配置环境变量

```bash
cp .env.example .env
```

用编辑器打开 `.env`，重点核对以下几项（与 vLLM 实际一致）：

```env
VLLM_BASE_URL=http://127.0.0.1:8000/v1
VLLM_API_KEY=vllm_api_key_12345
VLLM_MODEL=Qwen3.6-35B-A3B
VLLM_CONTEXT_WINDOW=32768
```

**如何确认模型名？** 在 vLLM 启动后执行：

```bash
curl -s -H "Authorization: Bearer vllm_api_key_12345" \
  http://127.0.0.1:8000/v1/models | python3 -m json.tool
```

把返回 JSON 里 `"id"` 的值填到 `VLLM_MODEL`。

### 第 5 步：启动 vLLM（单独一个终端）

```bash
# 示例，请按你机器上真实的模型路径/名称修改
vllm serve Qwen3.6-35B-A3B \
  --host 0.0.0.0 \
  --port 8000 \
  --api-key vllm_api_key_12345
```

保持此终端**不要关闭**。看到 `Uvicorn running` 或 `Application startup complete` 类似字样即可。

### 第 6 步：启动 KB Agent（再开一个终端）

```bash
cd sass    # 进入项目目录
source .venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload
```

看到：

```text
INFO:     Uvicorn running on http://0.0.0.0:8080
INFO:     Application startup complete.
```

表示启动成功。

### 第 7 步：自检

```bash
curl http://127.0.0.1:8080/health
```

应返回：`{"status":"ok"}`

## 五、日常使用（三种方式）

### 方式 A：网页界面（推荐给新手）

浏览器打开：

**http://127.0.0.1:8080**

1. **上传文档**：左侧选文件（支持 txt、md、pdf、Word `.docx`、Excel `.xlsx`/`.xls`、csv、pptx）→ 点击上传  
2. **文字入库**：在同一卡片下方粘贴文字 → 可选填标题 →「存入知识库」  
3. **提问**：右侧输入问题 → 发送  
4. 下方会显示回答和引用来源  

### 方式 B：API 文档页（Swagger）

**http://127.0.0.1:8080/docs**

- `POST /upload`：Try it out → 选文件 → Execute  
- `POST /chat`：填 `{"message": "你的问题"}` → Execute  

### 方式 C：命令行 curl

```bash
# 上传文件
curl -X POST "http://127.0.0.1:8080/upload" \
  -F "file=@/path/to/你的文档.pdf"

# 文字入库
curl -X POST "http://127.0.0.1:8080/ingest/text" \
  -H "Content-Type: application/json" \
  -d '{"content": "要入库的文字内容", "title": "可选标题"}'

# 提问
curl -X POST "http://127.0.0.1:8080/chat" \
  -H "Content-Type: application/json" \
  -d '{"message": "文档里写了什么？"}'
```

## 六、数据存在哪里？

| 内容 | 位置 | 说明 |
|------|------|------|
| 上传的原始文件 | `data/uploads/` | 文件名形如 `uuid_原文件名` |
| 向量索引 | `chroma_db/` | 自动生成，勿手动改 |
| Embedding 模型 | `models/bge-small-zh-v1.5/` | 预下载脚本生成 |

**对话记录不会保存**，只保存知识库文档和向量。

**清空知识库：**

1. 停止 uvicorn（Ctrl+C）  
2. 删除 `chroma_db/` 和 `data/uploads/` 里除 `.gitkeep` 外的文件  
3. 重新启动服务

## 七、远程服务器 / Cursor 开发注意

若代码跑在远程机器，浏览器在本机：

1. 在 Cursor **Ports** 面板转发 **8080**（和需要的 **8000**）  
2. 用 Cursor 提供的转发地址访问 `http://127.0.0.1:8080`  

若只绑定 `127.0.0.1` 且未做端口转发，本机浏览器无法直接访问远程服务。

## 八、常见问题速查

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| `/docs` 白屏 | Swagger 的 CDN 被墙 | 本项目已用本地 `static/`，确认能访问 `/static/swagger-ui-bundle.js` |
| 上传报 `Network is unreachable` | Embedding 模型未下载 | 执行第三节「第 3 步」镜像下载脚本 |
| 问答报 `Unknown model 'Qwen...'` | 模型名校验（旧版） | 拉取最新代码，确认使用 `VLLMOpenAI`；重启服务 |
| 问答报连接 LLM 失败 | vLLM 未启动或端口错 | 检查 8000 端口、`VLLM_API_KEY`、`.env` |
| 上传成功但答非所问 | 问题与文档无关或库为空 | 用文档里的原话提问；确认 `documents_indexed >= 1` |
| 根路径 `/` 404 | 旧版本无首页 | 更新代码，访问 `/` 应为网页界面 |
